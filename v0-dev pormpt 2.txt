# V0.DEV PROMPT — Role-Specific Registration Forms
## Registration Screen — Different Fields Per Entity Type

---

## CONTEXT

The Registration screen appears **after** the user selects their entity type on the "Who are you?" screen.

The selected entity type is passed as a prop/state to the Registration screen.
Based on the entity type, the form must **dynamically show different fields**.

There is NO single universal registration form.
There are **5 distinct form variants** — one per entity type.

---

## FLOW

```
Welcome Screen
      ↓
  [Sign Up]
      ↓
"Who are you?" Screen
  [Sponsor] [CRO] [SMO] [Site/Hospital] [Patient]
      ↓
Registration Screen
(form fields change based on selection above)
      ↓
OTP Verification
      ↓
Password Creation
      ↓
Account Created ✓
```

---

## SIGN IN — NO ENTITY SELECTION NEEDED

- Sign In screen is a **single universal screen** for ALL roles
- Just Email/Phone + Password
- System detects the role from the account and routes to the correct dashboard automatically
- Routes:
  - Sponsor / CRO → Sponsor Dashboard
  - SMO / Site / PI → Investigator Dashboard
  - Patient → Patient Dashboard
  - Platform Admin → Admin Dashboard

- Shown below the "Registration" title in the header

- Shows selected entity name: "Sponsor" / "CRO" / "SMO" / "Site/Hospital" / "Patient"

### Form Field Specs
- Label: 13px, `#475569`, above the input
- Input: white bg, `#E2E8F0` border, 12px radius, 48px height, 14px text `#0F172A`
- Focus state: `#2563EB` border (2px)
- Error state: `#DC2626` border + red error message below (13px)
- Required fields: asterisk (*) after label

### Phone Number Field
- Left section (prefix): gray-100 bg, `+91` text, border-right, not editable
- Right section: number input, no left border

### Organization Name Autocomplete
- As user types, show a white dropdown card below the input
- Dropdown items: org name + org type badge
- Tap item to fill the field and close dropdown
- Show "Not found? Continue typing your org name" at bottom of dropdown

### Role Toggle (Sponsor/CRO/SMO/Site only)
- Two buttons side by side, equal width, rounded-lg
- Active: `#0D1B3E` bg, white text
- Inactive: white bg, `#E2E8F0` border, `#64748B` text
- Only one can be active at a time

### Checkbox Style
- 20×20px square, rounded-sm
- Unchecked: white bg, `#CBD5E1` border
- Checked: `#0D1B3E` bg, white checkmark icon
- Tap anywhere on the label row to toggle

### Submit Button States
| State | Background | Text | Condition |
|-------|-----------|------|-----------|
| Disabled | `#CBD5E1` | `#94A3B8` | Required fields empty or checkbox unchecked |
| Enabled | `#0D1B3E` | White | All required fields filled + checkbox(es) ticked |
| Loading | `#0D1B3E` + spinner | "Submitting..." | While API call in progress |

## IMPLEMENTATION NOTES FOR V0

```jsx
// Pseudo-code structure
const RegistrationScreen = ({ entityType }) => {
  // entityType = "sponsor" | "cro" | "smo" | "site" | "patient"
  
  const isSMO = entityType === "smo";
  const isPatient = entityType === "patient";
  const isOrgUser = !isPatient; // sponsor, cro, smo, site

  return (
    <>
      {/* Common to all */}
      <FullNameField />
      
      {/* Org users only */}
      {isOrgUser && <DesignationField />}
      
      <EmailField />
      <PhoneField />
      
      {/* Org users only */}
      {isOrgUser && <OrgNameField />}
      {isOrgUser && <OrgAddressField />}
      {isOrgUser && <RoleToggle />}
      
      {/* SMO only */}
      {isSMO && <AddSitesSection />}
      
      {/* Patient only */}
      {isPatient && <DateOfBirthField />}
      {isPatient && <GenderDropdown />}
      {isPatient && <LanguageDropdown />}
      
      {/* Consent / Terms */}
      {isPatient ? <PatientConsentCheckboxes /> : <SingleTermsCheckbox />}
      
      <SubmitButton />
    </>
  );
};
```

## VALIDATION RULES

| Field | Rule |
|-------|------|
| Full Name | Required, min 2 chars |
| Email | Required for all, valid email format, duplicate check |
| Phone | Required, 10 digits (after +91 prefix) |
| Organization Name | Required for org users |
| Role | Required for org users, must select one |
| Site Name (SMO) | Required for each site card added |
| Date of Birth (Patient) | Required, must be valid date, user must be 18+ |
| Gender (Patient) | Required, must select one option |
| Language (Patient) | Required, must select one option |
| Terms checkbox | Must be checked before Submit enabled |
| Both consent checkboxes (Patient) | BOTH must be checked before Submit enabled |

## BULK ACTION MODE (long-press to activate)

When in bulk select mode:
- App bar changes: **"X selected"** (left) + **"Cancel"** (right, blue text)
- All cards show checkboxes
- Bottom action bar slides up (64px, white, shadow):

```
┌─────────────────────────────────────────┐
│ [Select All]   [🗑 Delete]  [🔕 Stop]   │
└─────────────────────────────────────────┘
```

- **Select All:** selects all visible notifications
- **Delete (red):** permanently deletes selected notifications (confirm dialog)
- **Stop (amber):** stops future notifications of the same type as selected

**Delete Confirmation Dialog:**
- "Delete X notifications?"
- "This cannot be undone."
- **[Cancel]** outlined | **[Delete]** red filled

## (Navigated from Profile menu → Change Password)

## APP BAR
- Back arrow → Profile & Settings
- Title: **"Change Password"**

## CONTENT

```
Current Password*
[••••••••              👁]

New Password*
[                      👁]

Password Strength:
[■][■][□][□][□]  Weak

Rules checklist:
✗  Minimum 8 characters
✗  Uppercase letter (A-Z)
✗  Lowercase letter (a-z)
✗  Numeric character (0-9)
✗  Special character (!@#$%...)

Confirm New Password*
[                      👁]
✗ Passwords do not match
```

- Strength bar: 5 segments
  - 1-2 filled = red = "Weak"
  - 3 filled = amber = "Medium"
  - 4-5 filled = green = "Strong"
- Rules: red X icon when not met, green check icon when met
- Match indicator below confirm field: red "✗ Passwords do not match" / green "✓ Passwords match"
- Eye icon toggles visibility on all 3 fields independently

**[Update Password]** full-width navy button
- Disabled: until current password filled + all 5 rules met + passwords match
- Success: green toast "Password updated successfully" + auto-navigate back to Profile

2. Tap **[Log Out]** →
   - Show brief loading spinner
