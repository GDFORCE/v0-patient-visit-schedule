## CHANGE 1 — MEDICATION "CONFIRM TAKEN" TICK BEHAVIOR
### (Dashboard → Medication section)

---

### CURRENT PROBLEM
The medication card on the Patient Dashboard shows a plain checkbox "Confirm taken" that does nothing visually meaningful when clicked.

### REQUIRED BEHAVIOR
Make it work **exactly like a to-do/task app** — when the patient ticks the checkbox, the medication animates out and the list updates dynamically.

---

### MEDICATION CARD — DEFAULT STATE

```
┌─────────────────────────────────────────┐
│  💊  Metformin 500mg        ☐ Confirm   │
│      🕐 8:00 AM               taken     │
└─────────────────────────────────────────┘
```

- Pill icon in teal circle (left)
- Drug name: 15px bold `#0F172A`
- Time: 13px `#64748B` with clock icon
- Checkbox (right): 22×22px, unchecked = white bg + `#CBD5E1` border

---

### WHEN PATIENT TAPS THE CHECKBOX — ANIMATION SEQUENCE

**Step 1 (0ms):** Checkbox fills instantly with teal `#0D9488` + white checkmark appears

**Step 2 (200ms):** Card background transitions to `#F0FDF4` (green-50)

**Step 3 (300ms):** Strikethrough line animates across the drug name (left to right, 300ms)

**Step 4 (600ms):** Small "✓ Logged at 8:03 AM" timestamp fades in below the drug name (green text, 12px)

**Step 5 (1200ms):** Card smoothly slides up and collapses out of view (height: auto → 0, opacity: 1 → 0, 400ms ease-in)

**Step 6:** Next pending medication slides up to fill the space (if any remaining)

---

### AFTER TICK — DYNAMIC STATES

#### State A: More medications remaining
Show the next pending medication card immediately after the completed one collapses:

```
Medication                          2/3 taken
─────────────────────────────────────────────
💊  Aspirin 75mg            ☐ Confirm taken
    🕐 2:00 PM
```
- Progress counter updates: "1/3 taken" → "2/3 taken"
- Counter shown as small gray text right-aligned on the section header

#### State B: All medications taken for the day
When the LAST medication is ticked and collapses, replace the medication section with:

```
┌─────────────────────────────────────────┐
│                                         │
│   ✅  (large teal checkmark, 32px)      │
│                                         │
│   All done for today!                   │  ← 15px bold #0F172A
│   No more medications required          │  ← 13px #64748B
│   Great job keeping up with your        │
│   schedule 💪                           │
│                                         │
└─────────────────────────────────────────┘
```

- Card: white bg, `#0D9488` left border (4px), rounded-xl, 16px padding
- Teal checkmark circle (40px): `#D1FAE5` bg + `#0D9488` check icon
- This state persists for the rest of the day
- Next day: resets to show tomorrow's medications as pending

#### State C: No medications scheduled today
If the patient has no medications at all for today:

```
┌─────────────────────────────────────────┐
│   💊  No medications today              │
│       Enjoy your medication-free day!   │
└─────────────────────────────────────────┘
```
- Gray pill icon + gray text
- Subtle gray border card

---

### UNDO FUNCTIONALITY
After ticking (before card collapses, during the 1200ms animation):
- A small **"Undo"** text link appears bottom-right of the card (blue, 13px)
- Tapping Undo within this window reverses the animation and unchecks

After card collapses: no undo available (medication is logged)

---

### MEDICATION SECTION HEADER UPDATE
```
Medication                    [1/3 taken]
```
- Section title "Medication" (left, 16px bold)
- Progress fraction right-aligned: "1/3 taken" → updates live → "All done ✓" when complete
- "All done ✓" shown in teal when all complete

## CRITICAL DESIGN RULES FOR THIS SCREEN

1. Red `#DC2626` is ONLY used for:
   - Status badges (Overdue)
   - Overdue numbers in KPI cards
   - "X days overdue" text labels
   - Protocol Deviation submit button (regulatory, irreversible)
   - Never used for action buttons that update visit status

2. Amber `#F59E0B` is used for:
   - "Review & Update" CTA on overdue cards
   - Warning banners
   - Screening status badges

3. Purple `#7C3AED` is used exclusively for:
   - eCRF-related elements (pending badge, eCRF section, notifications)
   - Creates a distinct visual language patients and PIs learn quickly

4. Visit window dates MUST appear on every visit card everywhere in the app —
   dashboard cards, patient list cards, calendar blocks, visit detail header.
   This is a GCP compliance requirement, not optional.

5. The bottom nav keeps 5 tabs: Dashboard / My Patients / Calendar / Notifs / Me.
   Do NOT add or remove tabs from this set for the PI role.

Build all screens in a single React component file with internal state routing.
Every button must navigate or trigger a visible state change.
Include the bottom nav on all 5 main tab screens, hide it on all detail/form screens.

The Registration screen appears **after** the user selects their entity type on the "Who are you?" screen.
